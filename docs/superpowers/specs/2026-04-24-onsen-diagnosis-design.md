# 温泉診断アプリ 設計仕様書

> 作成日：2026-04-24
> 事業：日本温泉（MyBusiness）
> プロジェクト：TechVillage/onsen-diagnosis-app

---

## ゴール

7問の診断クイズを通じて、ユーザーの温泉スタイルタイプを判定し、タイプに合った温泉地3件を提案する。結果ページから楽天トラベル・じゃらん等のアフィリエイト予約リンクと自社Webメディア記事へ誘導する。

---

## アーキテクチャ

### 技術スタック

- **フレームワーク**: Next.js（App Router）
- **スタイル**: Tailwind CSS v4（CSS-first、`@theme` in `globals.css`）
- **テスト**: Jest + ts-jest
- **デプロイ**: Vercel

### カラーテーマ

```css
/* globals.css の @theme に定義 */
--color-onsen-dark:   #7b4f2e;   /* メイン茶 — ヘッダー・ボタン */
--color-onsen:        #c8956c;   /* アクセント — 選択状態・プログレスバー */
--color-onsen-light:  #f5ead8;   /* 背景ベージュ — ページ背景 */
--color-onsen-white:  #fffdf7;   /* カード白 — カード背景 */
```

### ページ構成

```
/               → TOPページ（app/page.tsx）
/quiz           → クイズページ（app/quiz/page.tsx）
/result?type=X  → 結果ページ（app/result/page.tsx）
```

### データフロー

```
TOPページ
    ↓ 「診断スタート」ボタン
クイズページ（7問 × 1問ずつ表示）
    ↓ 全問回答後にスコアを計算
    ↓ sessionStorage に { type: 'T'|'B'|'S'|'K' } を保存
      キー名: 'onsen_quiz_result'
    ↓ router.push('/result?type=T')
結果ページ
    ↓ searchParams.type でタイプを取得（sessionStorageは使わない）
    ↓ タイプ定義から表示コンテンツを生成
    ↓ アフィリエイトリンク + 記事リンクを表示
```

## 診断タイプ定義（4タイプ）

| キー | タイプ名 | キャッチコピー | 特徴 |
|---|---|---|---|
| T | とろける湯タイプ | 疲れた体を芯から溶かす湯へ | 疲労回復・のんびり・長湯派 |
| B | 美肌の湯タイプ | 湯上がりの肌で、もう一度輝く | 美容・スキンケア・女子旅 |
| S | 秘湯探訪タイプ | 誰も知らない絶景の湯を探して | 絶景・秘境・非日常体験 |
| K | 活湯タイプ | 温泉で体をリセット、明日へ走る | 健康増進・朝活・リフレッシュ |

### タイプ別おすすめ温泉地（各3件）

**T（とろける湯）**
1. 草津温泉（群馬）— 日本三名泉。強酸性の湯で疲れをごっそり抜く
2. 別府温泉（大分）— 日本一の湧出量。多彩な泉質でじっくり癒される
3. 指宿温泉（鹿児島）— 砂むし温泉体験。体の芯まで温まる独自の文化

**B（美肌の湯）**
1. 玉造温泉（島根）— 日本最古の美肌の湯。ぬるっとしたアルカリ性単純泉
2. 嬉野温泉（佐賀）— とろとろ感日本一と称される美肌湯。温泉豆腐も名物
3. 湯布院温泉（大分）— 由布岳の麓の上質な湯と洗練された街並み

**S（秘湯探訪）**
1. 乳頭温泉郷（秋田）— 7つの宿が点在する秘湯の聖地。乳白色の湯が幻想的
2. 黒川温泉（熊本）— 山間の露天風呂めぐりが体験できる隠れ里
3. 登別温泉（北海道）— 地獄谷の迫力と多彩な泉質。大自然の中の非日常

**K（活湯）**
1. 有馬温泉（兵庫）— 日本最古の温泉。金泉・銀泉の二種類の泉質でリセット
2. 道後温泉（愛媛）— 日本書紀にも登場。歴史と現代が交差する活力の湯
3. 下呂温泉（岐阜）— 日本三名泉。肌あたりの柔らかい重曹泉で体をリフレッシュ

## クイズ設計（7問）

各選択肢はひとつのタイプに3点を付与。7問の合計点が最も高いタイプが診断結果になる。同点の場合は T > B > S > K の優先順で決定。

### Q1. 体の声を聞いてみよう

> 旅の前日夜、あなたの体はどんなサインを出していますか？

| 選択肢 | 加点 |
|---|---|
| 肩も腰もガチガチ。とにかく休みたい | T+3 |
| 最近、肌の乾燥やくすみが気になる | B+3 |
| 元気なのに、なぜかもっと活力が欲しい | K+3 |
| 毎日同じ景色に飽き飽きしている | S+3 |

### Q2. 温泉に何を求める？

> あの湯船に浸かったとき、何を感じたいですか？

| 選択肢 | 加点 |
|---|---|
| 疲れが溶けていくような、深い脱力感 | T+3 |
| 湯上がりに鏡を見てうれしくなる肌 | B+3 |
| 体の内側から元気が沸いてくる感覚 | K+3 |
| 「こんな場所があったのか」という感動 | S+3 |

### Q3. 好きな泉質は？

> 湯の個性で選ぶなら、どれに心が動く？

| 選択肢 | 加点 |
|---|---|
| 硫黄の香りがする、力強い白濁の湯 | T+3 |
| とろとろ・すべすべの美肌系アルカリ泉 | B+3 |
| 炭酸・塩化物系のパワフルな効能泉 | K+3 |
| 透明でクリアな源泉かけ流し | S+3 |

### Q4. 旅のスタイルは？

> 温泉旅行中、あなたはどう過ごしたい？

| 選択肢 | 加点 |
|---|---|
| 宿でひたすらのんびり。何もしないが最高 | T+3 |
| 温泉地の街をぶらぶら、お土産・スイーツも楽しむ | B+3 |
| 早起きして朝風呂、体を整えて一日をスタート | K+3 |
| 秘湯や絶景を求めて積極的に動き回りたい | S+3 |

### Q5. 誰と行く？

> 旅の相手が、温泉の楽しみ方を変える。

| 選択肢 | 加点 |
|---|---|
| 一人でじっくり、自分だけの時間 | T+3 |
| パートナーと二人、特別な時間を共有 | B+3 |
| 友人と賑やかに、笑いながら | K+3 |
| 家族みんなで、思い出をつくりに | S+3 |
<!-- 注：家族旅行は「遠出・非日常体験」と結びつけることでS寄りと定義。他の3タイプが他の選択肢で使用済みのため --> 

### Q6. どんな宿に泊まりたい？

> 宿のタイプも、温泉旅の大事な要素。

| 選択肢 | 加点 |
|---|---|
| 老舗の高級旅館。おもてなしと料理を堪能 | T+3 |
| 露天風呂付き客室で、誰にも気兼ねなく | B+3 |
| 活気ある温泉街の中の宿。街ごと楽しむ | K+3 |
| 山奥のこぢんまりした宿。電波も届かない | S+3 |

### Q7. どのくらい遠くまで行ける？

> 旅の距離感で、向かえる温泉地が変わる。

| 選択肢 | 加点 |
|---|---|
| 2〜4時間。少し遠くまで行ける | T+3 |
| 日帰りでも十分。サクッと行きたい | B+3 |
| 近場（2時間以内）でいい | K+3 |
| どこでもOK。遠ければ遠いほどいい | S+3 |

## ページ仕様

### TOPページ（`app/page.tsx`）

- ヘッダー：サービス名「日本温泉 診断」
- メインコピー：「あなたにぴったりの温泉地、見つけよう。」
- サブコピー：「7問に答えるだけ。約1分で診断完了。」
- CTAボタン：「診断スタート →」→ `/quiz` へ遷移
- 背景：`bg-onsen-light`（ベージュ）

### クイズページ（`app/quiz/page.tsx`）

- 1問ずつ表示（カード形式）
- 上部にプログレスバー（例：問3/7）
- 質問文（シナリオ調）+ 4択ボタン
- 選択後に即次の問へ遷移（確認ボタンなし）
- 7問終了後にスコア計算 → sessionStorage 保存 → `/result?type=X` にリダイレクト

### 結果ページ（`app/result/page.tsx`）

- `async` コンポーネント、`searchParams: Promise<{type?: string}>` を受け取る
- 不正な type 値は `T` にフォールバック
- 表示内容：
  1. タイプバッジ（キー1文字 + タイプ名）
  2. キャッチコピー
  3. 特徴説明（2〜3文）
  4. おすすめ温泉地カード × 3件（温泉地名・都道府県・一言説明）
  5. 各カードに「予約する」ボタン（アフィリエイトリンク）と「記事を読む」ボタン（自社記事リンク）
  6. SNSシェアボタン（X・LINE）

## テスト方針

- `lib/scoring.ts` のスコア計算・タイプ判定ロジックをJestでユニットテスト
- 7問すべてT選択 → T判定、7問すべてB選択 → B判定 を確認
- 同点ケース（例：T=9, B=9）→ T優先で判定されることを確認
